Voyage Log API

A read-only JSON API for a single vessel's published voyage log — diary entries, GPS track, and the latest telemetry. The default page at /logs/<handle> is built on this exact API; you can build your own front-end against it however you like.

Endpoint

GET  https://<your-project>.supabase.co/functions/v1/voyage-log?handle=<handle>&key=<key>

The exact URL — pre-filled with your handle and key — is shown in the Thalassa app under Settings → Voyage Log → API access.

Query parameters

Param Description
handle The vessel's public handle — a slug of the vessel name, e.g. serene-summer.
key The vessel's publishable API key. It is an identifier — not a secret — so it is safe to ship in a public web page. It gates rate-limiting and can be rotated by toggling the log off and on in the app.

Response

200 OK with a JSON body. Cache-Control: public, max-age=60 — the underlying data only moves every few minutes, so caching for a minute is plenty. CORS is open (Access-Control-Allow-Origin: *), so you can call it directly from any browser front-end.

Fields

Field Notes
vessel Vessel identity. model may be null.
entries Published diary entries, newest first (max 200). Only entries the skipper explicitly published appear here — the private diary is never exposed. latitude/longitude may be null if the entry has no pin.
track GPS breadcrumb for the configured window (default 30 days), oldest first. Numeric fields may be null for older fixes.
telemetry The most recent track point, surfaced for a "live" ribbon. null if the vessel has no track yet.

Errors

Errors return a JSON body of the shape { "error": "…" } with one of:

Status Meaning
400 handle or key missing from the request.
401 The key does not match the handle.
403 The vessel exists but its voyage log is switched off.
404 No vessel with that handle.

Example

const res = await fetch(
  'https://your-project.supabase.co/functions/v1/voyage-log' +
  '?handle=serene-summer&key=YOUR_PUBLISHABLE_KEY'
);
if (!res.ok) throw new Error((await res.json()).error);
const log = await res.json();

console.log(`${log.vessel.name} — ${log.entries.length} entries`);
if (log.telemetry) {
  console.log(`Now: ${log.telemetry.sog} kt, baro ${log.telemetry.baro} hPa`);
}
Building something with this? It's the same data the official renderer uses — feel free to lay it out however suits you. The skipper controls what's published and can switch the whole log off at any time.
